# GeneBrawl — Подробное объяснение всех файлов

> Мод для Brawl Stars на основе Frida. Написан на TypeScript, компилируется через Webpack в один bundle.js,
> который инжектируется в процесс игры через Frida.

---

## 🔑 Как это всё работает в принципе

Frida — это инструмент динамической инструментации. Он позволяет:
- **Interceptor.attach** — перехватить вызов функции и выполнить свой код до/после неё
- **Interceptor.replace** — полностью заменить функцию своей
- **NativeFunction** — вызвать любую функцию из игрового бинаря по адресу
- **NativePointer** — читать/писать в любую область памяти процесса

Все адреса функций (вида `0x6C5F50`) — это офсеты от начала `libg.so` (Android) или `laser` (iOS).

## 📁 src/index.ts — Точка входа

**Что делает:** Главный файл, с которого всё начинается. Экспортирует `rpc.exports.init`, который Frida вызывает при запуске скрипта.

**Порядок инициализации:**
1. `Configuration.load()` — загружает настройки из файла `settings.json`
2. `setupGame()` — патчит FPS, версию, шрифты, GUI, темы
3. `setupCustomAssets()` — загружает кастомные ресурсы мода (иконки, картинки)
4. `setupNetwork()` — патчит сетевые сообщения
5. `setupAllianceHooks()` — патчит альянсовый чат и чат-команды
6. `setupAvatarHooks()` — патчит аватар игрока
7. `setupHomeHooks()` — патчит главный экран
8. `setupBattleHooks()` — патчит боёвку (ввод, HUD, игроки)
9. `setupTeamHooks()` — патчит команды

**Затрагивает:** все остальные файлы проекта.

## 📁 src/libs/Libg.ts — Доступ к игровому бинарю

**Что делает:** Базовый класс для работы с бинарём игры (`libg.so` на Android, `laser` на iOS).

**Ключевой метод:**
```typescript
Libg.offset(android_arm64_offset, ios_offset)
```
Возвращает абсолютный адрес функции в памяти = `base_address + offset`.
Если офсет равен `0` или `-1` — значит он ещё не найден/не реализован.

**Пример:**
```typescript
const BattleScreen_enter = new NativeFunction(
    Libg.offset(0x6C5F50, 0x25F024), 'void', ['pointer']
);
// Android: libg.so + 0x6C5F50
// iOS:     laser   + 0x25F024
```

**Затрагивает:** почти каждый файл, где есть NativeFunction.

## 📁 src/libs/Libc.ts — Системные функции

**Что делает:** Обёртки над стандартными функциями C (`malloc`, `free`, `open`, `read`, `mkdir` и т.д.).

**Зачем нужно:**
- `Libc.malloc(n)` — выделить n байт памяти (для передачи строк/структур в нативные функции)
- `Libc.free(ptr)` — освободить память
- `Libc.getaddrinfo` — используется в ServerConnection для перехвата DNS-запросов
- `Libc.getSystemProperty` — читает Android-свойства (`ro.product.model` и т.д.)
- `Libc.sysctlbyname` — читает iOS-системные параметры

## 📁 src/utils/Armceptor.ts — Патчинг машинного кода

**Что делает:** Обёртка над `Memory.patchCode` для удобного патчинга ARM64-инструкций.

**Методы:**
- `nop(address)` — заменяет инструкцию на NOP (ничего не делать)
- `ret(address)` — вставляет RET (немедленный возврат из функции)
- `bytes(address, array)` — записывает произвольные байты
- `jumpout(address, target)` — вставляет безусловный прыжок на другой адрес

**Применение:** отключение защит, пропуск проверок, изменение логики без полной замены функции.

## 📁 src/gene/Configuration.ts — Настройки мода

**Что делает:** Хранит все настройки мода как статические поля класса. Загружает и сохраняет их в `settings.json`.

**Как работает:**
1. При старте вызывается `Configuration.load()` — читает `settings.json`, парсит JSON, присваивает поля
2. При изменении настройки в debug-меню вызывается `Configuration.save()` — сериализует все поля в JSON и записывает файл
3. `NCoder.s2n/n2s` — псевдошифрование (в публичной версии просто возвращает строку как есть)

**Важные поля:**
- `useProxy/useStage` — перенаправление трафика
- `autoAim`, `autoDodge`, `autoUlti` — боевые автоматизации
- `skinChanger` — смена скинов (реализация вырезана)
- `showFPS`, `showBattlePing` и т.д. — HUD-оверлей
- `accountNames` — словарь `{тег: имя}` для локальной смены ников

## 📁 src/gene/networking/NetworkManager.ts — Сетевой клиент мода

**Что делает:** Позволяет моду отправлять запросы на внешний сервер `scutils.hpdevfox.ru:13337`.

**Как работает:**
1. `NCoder.s2n(req)` — "шифрует" запрос (в публ. версии просто оборачивает в объект с метаданными)
2. Открывает TCP-сокет через `Socket.connect()`
3. Отправляет JSON-пейлоад
4. Читает ответ до `}`
5. `NCoder.n2s(str)` — "расшифровывает" ответ

**Используется для:** проверки ключа активации (`Configuration.key`), получения данных о пользователях.

## 📁 src/gene/features/ — Основные фичи

### ChatCommandHandler.ts
**Что делает:** Перехватывает отправку сообщений в командный чат (`TeamChatMessage_encode`) и в чат альянса (`ChatToAllianceStreamMessage_encode`).

**Как работает:**
1. Если сообщение начинается с `/` — передаёт в `CommandsHandler.execute()` и НЕ отправляет на сервер
2. Если `Configuration.antiProfanity` включён — вставляет спец-байты между символами (обход фильтра)
3. Иначе — отправляет оригинальное сообщение

### GradientNickname.ts
**Что делает:** Заменяет ники определённых игроков (по тегу) на цветные градиентные версии.

**Как работает:**
- Содержит захардкоженный словарь `{тег → цветной ник}`
- При рендеринге ника игрока перехватывает функцию `LogicPlayerTitleData_getTitleTID`
- Если тег есть в словаре — подставляет градиентный ник с тегами `<cRRGGBB>текст</c>`
- Также обрабатывает кастомные имена из `Configuration.accountNames` (заданные через `/name`)

### SkinChanger.ts
**Что делает:** Должен менять скины персонажей. **Реализация удалена из публичной версии** — все методы пустые.

### UsefulInfo.ts
**Что делает:** Собирает и отображает информацию на HUD.

**Как работает:**
- Каждую секунду (`secondsSinceLastUpdate`) создаёт `DebugHudMessageCollector`
- Добавляет в него строки: FPS, пинг, время, время сессии, тики, команду
- Передаёт коллектор в `Debug.getDebugHud().setMessageCollector()`
- `frameCounter` увеличивается каждый кадр — так считается FPS

### TeamSpam.ts
**Что делает:** Спамит одним сообщением в командный чат с интервалом 50мс.

**Как работает:**
1. `start()` — вешает одноразовый перехват на следующее отправленное сообщение
2. Запоминает текст сообщения
3. Запускает `setInterval` который каждые 50мс создаёт новый `TeamChatMessage`, устанавливает текст и отправляет через `MessageManager.sendMessage()`
4. `end()` — очищает interval и detach'ит listener

### Braille.ts
**Что делает:** Конвертирует текст в символы Брайля и обратно.

**Как работает:** Использует два словаря: `engDict+rusDict → brailleDict`. При вызове `Braille.to(text)` каждый символ заменяется соответствующим символом Брайля. Теги `<c...>` игнорируются (regex-обработка).

### DVD.ts
**Что делает:** DVD-скринсейвер на главном экране. Иконка (коляска) прыгает по экрану.

**Как работает:**
- Наследует `MovieClip`
- `update()` вызывается каждый кадр, двигает объект на (`xSpeed`, `ySpeed`)
- `checkHitBox()` — если достигнута граница экрана, меняет знак скорости (отскок)
- `createOnStage()` добавляет на главный спрайт игры через `GameMain.getHomeSprite().addChild()`

### Verevka.ts
**Что делает:** Анимированный объект (верёвка), спускающийся сверху на главном экране.

**Как работает:**
- Стартует с `y = -1000` (выше экрана)
- В `update()` увеличивает `y` на 5 пока не достигнет y=120
- Добавляется/удаляется с главного спрайта аналогично DVD

### Hamster.ts + hamster/Data.ts
**Что делает:** Мини-игра кликер с хомяком внутри debug-меню.

**Как работает:**
- Наследует `DebugMenuBase`
- Хранит `save: HamsterSaveData` — клики, энергия, эволюция
- При клике: если есть энергия — добавляет `clicksPerClick` кликов, тратит энергию
- Энергия восстанавливается со временем (`encountEnergyRestorationAmount`)
- `HamsterData` — конфиг: эволюции, скорость восстановления энергии

### RGBA.ts
**Что делает:** Утилита для работы с цветами в формате `0xAARRGGBB` (int32).

**Методы:**
- `RGBA.color(r, g, b, a)` — создаёт цвет
- `RGBA.hex(number)` — разбирает цвет обратно
- Предустановленные цвета: `RGBA.red`, `RGBA.green`, `RGBA.purple` и т.д.

### LobbyInfo.ts
**Что делает:** Текстовый оверлей в правом верхнем углу главного экрана.

**Содержит:** Версия скрипта, окружение (DEV/PROD), пинг, платформа, ссылка на Telegram. Обновляется каждый кадр в `update()`, скрывается когда открыт попап или идёт загрузка.

### UserImagesManager.ts
**Что делает:** Управляет скачанными пользовательскими изображениями (для установки кастомных картинок в игре).

### HttpClient/index.ts + Response.ts
**Что делает:** HTTP-клиент на основе Frida `Socket`. Выполняет GET/POST запросы к внешним URL.

**Используется для:** скачивания пользовательских изображений, работы с внешним API.

### filesystem/index.ts
**Что делает:** Обёртка над файловой системой через Frida `File` API. Читает, записывает, создаёт директории.

## 📁 src/laser/client/ — Клиент игры

### GameMain.ts

### MessageManager.ts (430 строк)
**Что делает:** Центральный перехватчик всех сетевых сообщений игры.

**Как работает:**
1. Перехватывает `MessageManager_receiveMessage` — каждое входящее сообщение от сервера
2. Через `LogicLaserMessageFactory` создаёт TypeScript-обёртку нужного типа
3. Вызывает соответствующий обработчик (`onLoginOkMessageReceived`, `onStartLoadingMessageReceived` и т.д.)
4. `sendMessage(message)` — отправляет сообщение на сервер

**Обрабатываемые сообщения:**
- `LoginOkMessage` — сохраняет accountId, показывает debug-UI
- `LoginFailedMessage` — показывает причину ошибки
- `OwnHomeDataMessage` — данные главного экрана
- `StartLoadingMessage` — начало загрузки боя (применяет SkinChanger)
- `BattleEndMessage` — конец боя
- `PlayAgainMessage` — кнопка "играть снова"
- `UdpConnectionInfoMessage` — UDP-данные для боя
- `PlayerProfileMessage` — профиль запрошенного игрока
- `TeamStreamMessage` — сообщения в потоке команды

### LatencyManager.ts
**Что делает:** Управляет тестами задержки сети. Периодически запускает latency-тесты, хранит результаты.

**Методы:**
- `getBestLatencyDataString()` — возвращает строку с лучшим пингом для отображения в LobbyInfo
- `patch()` — перехватывает `LatencyTestResultMessage` для сохранения результатов
